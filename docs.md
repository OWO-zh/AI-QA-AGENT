# 工具协同 Agent 系统 技术设计文档
## 1. 项目概述
### 1.1 项目背景
企业内部信息分散在文档库、业务数据库、外部资讯平台等多个系统，员工获取完整信息需跨平台操作，效率低且易遗漏；通用大模型虽具备推理能力，但无法直接访问私有数据、存在幻觉问题，且直接对接内部数据存在安全风险。
### 1.2 建设目标
构建一个可自主决策信息来源、协同调度多类工具、整合分散信息输出统一回答的智能 Agent 系统，实现：
- 一个入口覆盖内部文档、业务数据、外部资讯三类信息
- 回答可溯源、可解释，降低大模型幻觉影响
- 数据安全隔离，敏感信息不流出企业内网
- 高可用容错设计，减少系统静默失效
### 1.3 核心验收指标
- 工具调用准确率 ≥ 90%
- 知识范围内回答准确率 ≥ 90%
- 系统静默失败率 ≤ 1%
- 支持知识库文件热更新与新工具快速扩展
## 2. 整体架构设计
### 2.1 架构总览
系统采用分层设计，自下而上分为：基础资源层、工具能力层、Agent 调度层、交互展示层。
- **基础资源层**：大模型服务、向量数据库、业务数据库、搜索 API
- **工具能力层**：将各类能力封装为标准 Function Calling 工具，统一输入输出格式
- **Agent 调度层**：基于 LangGraph 实现状态机调度，负责意图决策、工具执行、结果生成
- **交互展示层**：Streamlit 可视化界面，负责用户交互、状态展示、结果呈现
### 2.2 状态机工作流设计
基于 LangGraph 构建三节点有状态工作流，支持条件分支与状态持久化：
1. 决策节点（Decide Node）
    - 输入：用户问题 + 对话上下文
    - 能力：意图识别、工具选择、参数提取
    - 分支：无需工具直接回答 / 调用单工具 / 调用多工具并行执行
2. 执行节点（Execute Node）
    - 输入：决策节点输出的工具列表与参数
    - 能力：并行 / 串行执行工具、异常捕获、自动重试
    - 输出：结构化的工具返回结果
3. 生成节点（Answer Node）
    - 输入：原始问题 + 工具返回结果集合
    - 能力：信息整合、溯源标注、答案生成
    - 输出：带来源标注的最终回答
### 2.3 数据流走向
用户提问 → 会话上下文注入 → 意图决策 → 工具路由 → 工具执行（含容错重试）→ 结果聚合 → 答案生成 → 返回前端展示
## 3. 核心模块详细设计
### 3.1 决策节点
- 实现方式：通过系统 Prompt 约束模型输出结构化 JSON，包含工具名称、调用参数、调用说明
- 路由规则：
    - 问题涉及内部文档 / 制度 → 路由至知识库检索工具
    - 问题涉及业务数据统计 / 查询 → 路由至数据库查询工具
    - 问题涉及实时资讯 / 外部信息 → 路由至联网搜索工具
    - 复杂问题同时涉及多类信息 → 并行调度多个工具
- 优化手段：Few-Shot 示例引导，提升工具选择与参数提取准确率
### 3.2 工具执行模块
#### 3.2.1 RAG 知识库检索工具
- 分块策略：采用带重叠的递归字符分块，兼顾上下文完整性与检索精准度
- 向量存储：基于 FAISS 构建本地向量索引，支持增量更新
- 检索机制：相似度检索 + 重排序，返回 Top-K 最相关文档片段
- 拒答设计：设置相似度阈值，低于阈值则判定无匹配内容，触发拒答逻辑
#### 3.2.2 数据库查询工具
- 生成逻辑：大模型根据表结构 Schema 自动生成 SQL 查询语句
- 安全校验：内置 SQL 白名单校验，仅允许 SELECT 操作，禁止删改、高危函数
- 自纠错机制：SQL 执行报错时，将错误信息回传模型自动修正，最多重试 2 次
#### 3.2.3 联网搜索工具
- 基于 Tavily Search API 实现实时资讯检索
- 返回结果包含标题、摘要、来源链接，确保信息可追溯
- 支持关键词自动扩展，首次检索结果不足时自动优化关键词二次检索
#### 3.3 答案生成节点
- 统一整合多工具返回的结构化结果，按信息来源分段落组织答案
- 强制标注每条信息的来源（文档片段 / 数据库数据 / 搜索来源）
- 约束模型仅基于检索结果作答，无对应信息时主动说明，不编造内容
## 4. 工程化与可靠性设计
### 4.1 安全防护体系
- 数据安全：数据库查询与联网搜索物理隔离，内部数据不传递给外部 API
- SQL 安全：白名单语法校验 + 只读账号权限，防止注入与数据篡改
- 文件安全：上传文件仅允许 PDF/TXT 等文本格式，禁止可执行文件上传
- 会话隔离：基于 Session 实现用户级数据隔离，知识库与对话记录互不干扰
### 4.2 三级容错降级机制
1. **一级容错**：工具调用失败自动重试（最多 2 次），适配网络波动与接口限流
2. **二级容错**：检索结果为空时，自动扩展关键词 / 放宽匹配条件二次检索
3. **三级容错**：所有工具均无有效结果时，主动告知用户并给出查询建议，杜绝静默失败
### 4.3 全链路日志系统
- 分级日志输出：INFO/DEBUG/WARNING/ERROR 四级
- 全链路埋点：记录每轮对话的意图决策、工具调用、执行耗时、返回结果
- 日志持久化：本地文件存储，支持问题回溯与效果调优
### 4.4 会话状态管理
- 基于 Streamlit Session State 实现会话级状态存储
- 对话历史、知识库实例、工具状态均与会话绑定
- 支持手动清空对话，重置会话状态
## 5. 性能与效果验证
### 5.1 测试数据集
- 构建 1200 条标注测试集，覆盖单工具查询、多工具混合查询、无答案拒答三类场景
- 包含常规问题、边界问题、诱导性问题，覆盖常见使用场景

| 知识内回答准确率 | 91% |

## 6. 扩展开发指南
新增工具仅需三步：
1. 在 tools/ 目录下实现工具函数，定义标准输入输出
2. 补充 Function Calling 描述与参数 Schema
3. 在 LangGraph 工作流中注册工具，更新决策节点 Prompt
## 7. 部署说明
- 本地开发：直接通过 Streamlit 命令启动
- 容器化部署：提供 Dockerfile，支持一键构建镜像与部署
- 生产建议：接入 Redis 持久化会话、增加用户认证、配置接口限流
